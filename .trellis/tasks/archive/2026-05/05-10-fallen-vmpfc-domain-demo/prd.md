# Run Fallen vmPFC Domain Demo

## Goal

Run the existing vmPFC domain-pool, active-dimension inference, and first-pass description scoring workflow for the newly added `fallen` dataset, so we can inspect whether the inferred vmPFC dimensions are suitable before deciding whether to build a separate anxiety-specific prompt.

## What I Already Know

- The user added `fallen/` with `refined_description.md` and five GT CSV files under `fallen/gt/`.
- The GT CSV files contain second-level labels in the `情绪值` column and `视频时间(s)` spans approximately `0-6072s`.
- `fallen/refined_description.md` contains timestamped descriptions spanning approximately `0-6070s`.
- The user wants `target_region` to be `vmPFC` for this first pass.
- The first pass has produced a vmPFC prompt with 20 active dimensions.
- The user confirmed the next step should score `fallen/refined_description.md` with this vmPFC prompt.
- The raw description file contains non-description metadata, two zero-duration blocks, and one malformed timestamp fragment, so a parser-compliant scoring copy is required before running `score-descriptions`.

## Assumptions

- Use the maintained `brain_region_pipeline` CLI rather than adding new source code.
- Use `test_data/Schaefer2018_1000Parcels_17Networks_order (1).txt` as the atlas label input.
- Store generated artifacts under a dedicated `fallen/demo/` folder so they stay separate from raw dataset files.
- If `make-module-prompt --domain-pool` requires a confirmed pool, create a reviewed confirmed copy after inspecting the draft domain pool rather than overwriting the raw draft.

## Requirements

- Generate a vmPFC domain-pool draft for the `fallen` demo.
- Inspect the generated domain pool for relevance to vmPFC and the anxiety-like content in `fallen`.
- If the draft is structurally valid and relevant enough for this exploratory pass, mark a copy as confirmed and use it for active-dimension inference.
- Generate a vmPFC module prompt with concrete active dimensions guided by the confirmed domain pool.
- Summarize the inferred domains and dimensions for user review.
- Score all `fallen/refined_description.md` segments with `fallen/demo/vmpfc_module_prompt.json`.
- Generate a parser-compliant description copy before scoring, preserving valid timestamped blocks and recording any dropped invalid blocks.
- Store the first-pass scoring output in a dedicated subdirectory under `fallen/demo/`.

## Out of Scope

- No changes to `brain_region_pipeline` source code.
- No GT interval averaging or correlation analysis in this task.
- No new standalone anxiety prompt unless the domain/dimension review indicates it is needed and the user confirms a follow-up.

- Relevant command path: `uv run python -m brain_region_pipeline make-domain-pool`.
- Relevant command path: `uv run python -m brain_region_pipeline make-module-prompt --domain-pool`.
- Relevant command path: `uv run python -m brain_region_pipeline score-descriptions`.
